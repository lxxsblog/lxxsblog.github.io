数据结构是非常的重要的，以前没学的时候做题总是没有头绪，有了数据结构就会好很多。😊

NOIP-数据结构

二叉树

顺序存储

这个地方可以举个例子；当我们要存储时要按照完全二叉树的方式存储，也就是说空节点也要考虑到。
如果我们要存储一颗下图所示的树事，我们不能在数组中这样存储，而是把他补全成完全二叉树,详细看第二张图。

简单说就是当我们输入这颗树时不应该是RABCDEFGH 而是RABCDEFGH,在数组就对这些数据进行编码了；
注意这个很重要，这个不要和二叉树的生成算法搞混了，生成算法用的链式存储。

链式存储

二叉链表节点
图片表示：

**代码表示：

struct node{
    int date;
    node *lchild=NULL;
    node *rchild=NULL;
};

三叉链表就是在二叉链表的基础上加入了父母节点
构造过程和单链表相似，这个代码构造是不需要可以赋予空指针，因为它本身已经赋值好了。
在n个节点的二叉链表中，又n+1个空指针与：必定又2n个链域，除根结点外每个节点有且仅有一个双亲，所有只会有n-1个节点的链域存放着指针，只想非空子女节点。空指针树=链域-指针数；

遍历二叉树

D->根  L->左   R->右
先根遍历->DLR    中根遍历->LDR    后根遍历->LRD：一定是先左子树再右子树，变化的是根，也可为*序遍历；
重点（代码实现）:

void Traverse(bitnode &head){
    if(head=NULL) return;
    else{
    	printf("%d",head->date);//访问根节点
        Traverse(head->lchild);//递归访问左子树
        Traverse(head->rchild);//递归访问右子树
    }//先序中序后序知识把中间调换了而已！！！

}

层次遍历：有点像广度优先搜索；
就不写代码了，简单描述一下吧：首先初始化一个循环队列，将根节点指针放入队列，当队列不为空时，弹出队列中的结构指针本访问它的data域，访问他的孩子，如果又左孩子将其入队，如果又右孩子将其入队，重复循环；

建立二叉树算法

这个挺简单的，不过输入的时候要用一些特殊的符号表示其孩子节点为空；
比如先序遍历：ABC##DE#G##F###他的图像：

    ![Snipaste_2019-12-01_22-48-09.png](https://cdn.nlark.com/yuque/0/2019/png/432966/1575211710910-65626d72-43a2-43a8-a00a-1bbd27d50326.png)


代码实现：

void CreatBiTree(bitnode &head){//带值构造，会影响实际；
	scanf(&ch);
    if(ch=='#') head=NULL;//如果输入的字符是特殊字符，那么这节点为空；
    else{
        head=new bitnode;//为这个数据申请一个新节点；
        head->date=ch;//将节点的data域赋值为ch；
        CreatBitree(head->lchild);//构造他的左子树；
        creatBitree(head->rchild);//构造他的右子树；
    }//*序遍历只需要将else重点额内容调换即可；
    return;
}

双亲表示法

这个可以看一下2019CSP复赛的第二题，定义的时候只需要知道数据和双亲就可以。完全可以用结构数组来做，R的双亲为-1，其他节点的而双亲为双亲数据所在的下标。
找双亲容易找孩子难。

孩子链表表示法

有两部分：双亲节点数组+孩子链表
双亲结点数组：|数据|指向第一个孩子的指针|
孩子链表：|孩子在双亲结点数组中的下表|指向下一个孩子的指针|
代码实现：

struct childnode{
	int child;//存储下标；
    childnode *next;
};

struct fathernode{
	char date;
    childnode *first;
};

struct tree{
	fathernode[MAX];
    int n,r;//节点数和根结点数
};

找孩子容易，找双亲难；

孩子兄弟表示法

这个方法找孩子容易找父母也容易，可以理解为这个是把树转化成二叉树了。右节点是下一个兄弟，左节点一定是第一个孩子；也就是向右可以找兄弟，向左可以找孩子;
孩子兄弟节点：|date|fierstchild|nextbrother|

代码实现：

struct childbrotree{
	int date;
    childbortree *nextbro,*firstchild;
};

树，森林，二叉树的的转换

森林->树=》D(森林)//给森林的所有树加一个根节点
树->二叉树=》childbrotrr(树)把树转成二叉树
森林->二叉树=》森林->树+树->二叉树

哈夫曼树

概念
路径：从一个节点到另一个节点的路径树的路径长度：从树根到每一个节点的路径长度之和TL。节点数目相同的二叉树中，完全二叉树是路径最短的二叉树。TL=每个节点的根节点的路径长度之和；weight(权重)：树中结点赋予一个特殊意义的值，叫做这个节点的劝；节点的带权路径长度：从根节点到该节点之间的路径长度和该节点的权的乘积。树的带权路径长度：书中所有叶子节点的带权路径长度之和；WPL
哈夫曼树：WPL最小的树是哈夫曼树；满二叉树不一定是哈夫曼树；具有相同带权节点的哈夫曼树不唯一；
哈夫曼是构造算法：
1.构造森林全是树；2.选用两小造新树3.删除两小舔新人；4.重复2，3剩单树；
包含n棵树的森林要经过n1次合并才能形成哈夫曼树，共产生n-1个新结点
包含n个叶子结点的哈夫曼树中共有2n-1个结点。
哈夫曼树的结点的度数为0或2，没有度为1的结点。
